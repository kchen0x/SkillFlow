## 1. 导航与外壳

固定左侧边栏（宽 w-56）提供全局导航。

| 路由 | 图标 | 标签 |
|------|------|------|
| `/` | Package | 我的skills |
| `/tools` | Wrench | 我的智能体 |
| `/prompts` | FileText | 我的提示词 |
| `/sync/push` | ArrowUpFromLine | 推送到智能体 |
| `/sync/pull` | ArrowDownToLine | 从智能体拉取 |
| `/starred` | Star | 仓库收藏 |
| `/backup` | Cloud | 云备份 |
| `/settings` | Settings | 设置 |

- 当前路由：使用当前主题色的浅色块高亮，并配合克制的描边与层次阴影。
- 非当前路由：灰色文字，鼠标悬停时显示高亮。
- 侧边栏左上角：`SkillFlow` 标题文字左侧显示应用图标；图标会略高于标题文字以保证辨识度，并保持原始长宽比。
- 侧边栏右上角：**Languages** 语言快捷按钮，可在 **中文** 与 **English** 之间即时切换，并将偏好持久化到 `localStorage`。
- 侧边栏右上角：**调色盘** 主题快捷按钮，按 **Dark → Young → Light** 顺序即时轮换。
- 左下角 **意见反馈** 按钮：在默认浏览器中打开 GitHub 新建 Issue 页面。
- 窗口关闭行为：点击左上角关闭按钮时会直接结束当前 UI 进程，而不再只是隐藏主窗口。轻量 helper 会继续驻留在托盘/菜单栏，因此通过 `Show SkillFlow` 或再次启动应用都能重新打开一个新的窗口。
- 主页面刷新行为：路由切换时会按 `location.pathname` 重建页面子树，因此重新进入 **我的skills**、**我的智能体**、**我的提示词**、**仓库收藏** 等页面时，会再次拉取后端当前数据，而不需要在页面内手动刷新。
- 启动平滑策略：主界面准备好后，启动阶段的后台任务会错峰启动，而不是同时触发，因此首屏可交互的前几秒不容易和 Skill 更新检测、收藏仓库刷新、应用更新检查、Git 启动拉取一起争抢资源。
- 窗口重新激活行为：当窗口通过托盘/菜单栏再次显示，或长时间不活跃后重新回到前台时，SkillFlow 现在会继续保留当前路由页面树。用户会立即看到离开前的页面内容、滚动位置和进行中的交互，而不会再先看到“后台内存收缩中”占位页，也不会因为重新挂载路由而打断当前操作。
- 窗口启动尺寸：启动时会优先恢复保存在本地 `config_local.json` 里的最近一次窗口尺寸；如果用户还没有手动调整过，则会根据当前显示器尺寸自适应计算初始窗口，使用更适合桌面的默认大小，并在屏幕可用空间内居中显示。
- macOS 托盘行为：一个轻量 helper 会在顶部状态栏保留黑白状态栏图标。关闭主窗口会退出 UI 进程，但状态栏菜单仍然保留，可执行 `Show SkillFlow`、`Hide SkillFlow` 和 `Quit SkillFlow`；其中 `Show SkillFlow` 会重新拉起或聚焦 UI 窗口。
- Windows 托盘行为：一个轻量 helper 会继续驻留在系统托管区并显示应用图标。关闭主窗口会退出 UI 进程；点击托盘菜单仍可通过 `Show SkillFlow` 重新拉起或聚焦 UI，也可通过 `Exit` 一并退出 helper 和 UI。

## 2. 我的skills（主面板）

集中管理你的 Skill 库。

### 工具栏

| 控件 | 操作 |
|------|------|
| **搜索框** | 加宽显示，实时按技能名过滤（不区分大小写）；窗口较窄时工具栏会自动换行，保证按钮完整可见 |
| **排序切换** | 两个按钮在 **A-Z** 与 **Z-A** 间切换，按 Skill 名称首字母正序或逆序排列 |
| **更新**（RefreshCw） | 调用后端 `CheckUpdates()`；按规范化后的仓库来源 + 子路径聚合同一个 Git Skill，把已安装实例的 `SourceSHA` 与本地仓库缓存中同一子路径的最新 SHA 做比较，刷新可同步的 `LatestSHA` 与仅本地保存的 `LastCheckedAt`，为可更新卡片显示红点与“更新”操作，并在实例已是最新时清理过期更新标记。该按钮只检查，不会直接覆盖本地文件 |
| **批量删除**（CheckSquare） | 切换多选模式 |
| **导入**（FolderOpen） | 打开系统文件夹选择器 → `ImportLocal(dir)` |
| **自动更新**（ToggleLeft / ToggleRight） | 继续占用原来远程安装按钮的位置，但现在改成明显的开关态按钮：关闭时显示低强调的“开自动更新”，开启后切成高强调的“关自动更新”，并通过左右切换图标直接区分当前状态。切换成功后会显示确认提示；开启后仍然会在启动时或手动刷新收藏仓库后，自动更新匹配的已安装 Git Skill |

### 自动推送目标智能体

- 工具栏下方新增一条紧凑的单行选择区，把 **自动推送目标智能体** 标题和智能体按钮放在同一行，按钮样式与 **推送到智能体** 页面保持一致。
- 这组选择只保存在当前设备本地，并会在后续继续复用。
- 工具栏里的 **自动更新** 开关也只保存在当前设备本地，与自动推送目标智能体一起持久化。
- 这个按钮现在显示的是“下一步动作”而不是固定标题，并通过左右切换图标与高/低强调样式直接告诉你当前是开着还是关着。
- 勾选某个智能体后，会立即把当前 **我的skills** 里已有的 Skill 补推送到该智能体，不需要等到下一次导入。
- 任何新添加到 **我的skills** 的 Skill，在成功导入库之后都会自动复制到这些选中的智能体中；覆盖本地导入、从智能体拉取、以及仓库收藏导入这几条入口。
- 如果云端恢复把库内 Skill 同步到了当前设备，SkillFlow 会立刻对这些“新恢复”或“恢复后变成新版本”的 Skill 执行自动推送，目标仍是当前设备已勾选的智能体。
- 本地导入的自动推送仍是非破坏性的：如果目标智能体的 `PushDir` 中已经存在同名 Skill，SkillFlow 会跳过该目标，不会覆盖已有内容。
- 之后取消勾选某个智能体时，也不会删除之前已经推送过去的 Skill；如需移除，仍需用户在 **我的智能体** 页面或智能体目录里手动删除。

### 多选模式（点击"批量删除"激活）

| 控件 | 操作 |
|------|------|
| **全选 / 取消全选** | 切换当前过滤后所有 Skill 的选中状态 |
| **删除（n 个）**（Trash2，红色） | `DeleteSkills(ids)` — 未选择时置灰禁用 |
| **取消** | 退出多选模式并清空选择 |

### 分类侧边栏

- 列出所有分类；点击可过滤 Skill 网格。
- **"全部" 按钮** — 显示所有分类的 Skills。
- **拖拽目标区域** — 将 Skill 卡片拖到某个分类，即可移动至该分类。
- **拖拽高亮** — 拖动 Skill 卡片悬停到分类上方时，目标分类会以更明显的高亮状态显示。
- **右键上下文菜单**（每个分类）：
  - **重命名** — 显示内联文字输入框；回车确认，Esc 取消；调用 `RenameCategory()`。（`Default` 分类不可用）
  - **删除** — 空分类会直接调用 `DeleteCategory()` 删除；若该分类下仍有 Skill，则弹出提示框要求先清空该分类后再删除。（`Default` 分类仍不可删除）
- **新建分类**（底部 Plus 图标） — 显示内联文字输入框；回车或失焦确认，Esc 取消；调用 `CreateCategory()`。

### Skill 网格

- 布局：3 列，宽屏 4 列。
- 列表性能行为：进入页面时会优先读取本地派生快照来构建已安装 Skill 列表；页面切换动画也改成更轻量的淡入淡出，以减少切页卡顿。
- 大列表动画降级：当 **我的skills** 当前可见卡片超过 18 张时，会自动关闭逐卡错峰入场动画，改为直接渲染，避免大列表时的掉帧。
- 卡片状态条溢出处理：状态徽标和已推送智能体图标现在使用固定截断与 `+N` 溢出显示，不再对每张卡片做运行时宽度测量。
- **空状态** — 显示"没有找到 Skills"及使用提示。
- **Skill 右键菜单** — 包含"移动到其他分类"项（仅显示非当前分类）、删除，以及适用时的更新操作。
- **拖拽操作** — 将 Skill 卡片拖至侧边栏分类可移动；开始拖拽后，会有一个缩小版卡片跟随鼠标移动；命中左侧分类后，原卡片会收缩成一条细线，直到结束拖拽；将文件夹从操作系统文件管理器拖入窗口可直接导入。
- **窗口级拖拽遮罩** — 拖拽文件进入窗口时，显示半透明靛蓝色遮罩和"松开以导入 Skill"提示。
- **悬浮提示** — 鼠标悬停卡片 300 毫秒后出现（详见 [Skill 悬浮提示](#9-skill-悬浮提示)）。

### Skill 更新流转

- 工具栏的 **更新** 只负责把已安装的 Git Skill 与本地缓存仓库中的对应子路径做比较，并把卡片标记为“可更新”。
- 卡片上的 **更新** 才会真正从本地仓库缓存复制最新文件，覆盖 **我的skills** 中已安装的那一份副本，并同步刷新已经存在于各智能体 `PushDir` 中的同一 Skill 副本；同时会对当前勾选了“自动推送目标智能体”的智能体执行强制同步（不存在则新建，存在则覆盖为最新版本）。
- 当主面板中的 **自动更新** 开关处于开启状态时，启动后的收藏仓库刷新，以及你在 **仓库收藏** 页面执行的单仓库刷新或全部刷新，都会自动触发同一套已安装 Skill 更新流程。
- 自动更新只会为当前勾选了“自动推送目标智能体”的智能体创建或覆盖副本；未勾选的智能体会保留现有副本，并在该副本落后于 **我的skills** 时继续显示“可更新”。
- 卡片更新进行中时，该卡片会保持“更新”操作可见、禁止重复点击，并持续旋转 Refresh 图标直到请求结束。
- 主面板顶部还会显示临时状态提示条，明确告知当前正在更新、更新成功或更新失败，方便用户确认点击确实已触发操作。
- 基于缓存的检查会按应用内统一使用的 Git 逻辑主键聚合：规范化仓库来源 + 仓库子路径。
- 当多个已安装实例指向同一个逻辑 Git Skill 时，缓存中的 SHA 只读取一次，但每个实例仍会用自己的 `SourceSHA` 决定自己是否 `updatable`。
- 如果对应的本地仓库缓存不存在或尚未刷新，这里不会退回到直接请求 GitHub API；需要先通过正常的仓库缓存同步链路更新该 clone。

```text
[主面板工具栏 更新]
         |
         v
    CheckUpdates()
         |
         v
比较本地 SourceSHA 与缓存仓库子路径的最新 SHA
         |
         +--> 没有更新  -> 卡片保持不变
         |
         +--> 有更新    -> 写入 LatestSHA -> 显示红点 / 更新按钮

[Skill 卡片 更新]
         |
         v
 UpdateSkill(skillID)
         |
         v
从本地缓存 clone 复制该 skill 对应的仓库子路径
         |
         v
覆盖“我的skills”中的安装目录
         |
         v
刷新各智能体 PushDir 中已存在的同一 Skill 副本
         |
         v
SourceSHA = LatestSHA -> 清除更新标记
```

## 3. 推送到智能体

将 Skill 库中的 Skills 复制到外部智能体目录。

### 布局

- 采用与"我的skills"接近的双栏布局。
- 左侧边栏显示分类筛选：**全部** 加所有现有分类。
- 右侧显示目标智能体选择、搜索 + A-Z/Z-A 排序控件、推送模式按钮，以及当前分类范围内的 Skill 卡片网格。

- 页面间距与网格密度做了收紧，配合启动时自适应的窗口尺寸，在常见笔记本和桌面分辨率下，通常可以在首屏同时看到头部控件、Skill 列表和底部推送操作，而不是一打开就立刻需要滚动。
### 目标智能体选择

- 每个已启用的智能体对应一个切换按钮（图标 + 名称）。
- 可同时选择多个智能体。
- 当前激活的分类、智能体和范围按钮会使用更亮的主题底色、更浅的边框和更明显的发光，因此在深色主题下不需要额外符号也能清楚区分选中状态。

### 同步范围

推送行为由左侧分类筛选和右侧模式按钮共同决定：

| 模式 | 行为 |
|------|------|
| **手动选择** | 位于 **推送全部 / 推送当前分类** 左侧，且默认选中；以当前左侧分类范围作为候选列表，在卡片上逐个勾选，也可对当前列表执行全选 |
| **推送全部 / 推送当前分类** | 左侧选中 **全部** 时推送整个库；选中具体分类时只推送该分类 |

### 目录缺失检查

推送前，应用调用 `CheckMissingPushDirs()`。若目标智能体目录尚未创建，弹出确认对话框：

- 列出每个缺失的智能体名称及完整目录路径。
- **"创建并推送"** — 创建目录后继续推送。
- **"取消"** — 中止，不创建任何目录。

### 冲突处理

若目标目录中已存在同名 Skill，逐条弹出冲突对话框（详见 [冲突对话框](#101-冲突对话框)）。

### Skill 网格

- 这个页面的库内 Skill 卡片只展示推送决策真正相关的状态：哪些智能体的 `PushDir` 已经包含该逻辑 skill。
- 卡片会继续保留“来源”徽标，因此在决定推送时仍可直接区分该 Skill 是手动导入还是 Git 来源导入。
- 已推送智能体会用紧凑图标列表表示；数量过多时折叠为省略号，并支持悬浮查看完整列表。

### 底部操作栏

- **"开始推送（n）"** 按钮 — 未选智能体或数量为零时禁用；进行中显示"推送中…"。
- **"推送完成 ✓"** — 推送结束后绿色成功提示。

## 4. 从智能体拉取

将外部智能体目录中的 Skills 导入到你的库中。

### 布局

- 采用与“推送到智能体”一致的双栏布局。
- 左侧边栏列出全部分类，并直接作为导入目标分类选择器。
- 右侧显示来源智能体选择、扫描反馈、搜索 + A-Z/Z-A 排序控件、可选 Skill 网格和底部执行栏。

### 来源智能体选择

- 与推送页相同的切换按钮；切换智能体时重置已扫描列表。
- 当前导入目标分类与来源智能体沿用推送页相同的高亮底色、浅色边框与发光选中样式，因此在深色主题下也能更稳定地识别当前选择。

### 扫描

- **"扫描"** 按钮 — 调用 `ScanAgentSkills(agentName)`，递归搜索该智能体已配置的扫描目录中的 `skill.md` 文件。
- 本地智能体扫描会复用 **设置 → 通用** 中相同的深度限制（默认 `5`，保存范围 `1-20`）。
- 扫描中显示动效"扫描中…"。
- 扫描失败时显示**红色错误提示**；未发现 Skill 时显示**黄色警告提示**。
- 智能体扫描候选项会优先按逻辑主键去重和关联；只有在无法得到稳定主键时才会退回弱匹配，因此同名但内容不同的 Skill 不会再被直接合并。

### Skill 网格

- 扫描成功后显示。
- 搜索框会实时按名称过滤扫描结果。
- 排序切换按钮可在 **A-Z** 与 **Z-A** 间切换。
- 如果某个扫描项能关联到“我的skills”里已安装的 Skill，卡片会同时显示该已安装项的来源徽标，方便区分它原本是手动导入还是 Git 来源导入。
- 卡片仍会显示拉取决策需要的“是否已导入”状态；对尚未导入、也无法关联到本地安装记录的扫描项，不会强行猜测来源。
- 每次扫描后，所有 Skill 默认均为未选中。
- 可逐个选择，可对当前可见列表执行 **全选 / 取消全选**，也可通过与“全选”同样的方形切换按钮 **选择未导入**，一键勾选当前可见且尚未导入的 Skill。
- 选择状态和拉取冲突都按扫描到的路径跟踪，因此来自不同智能体目录的同名 Skill 也能独立处理。

### 底部操作栏

- **"开始拉取（n）"** 按钮 — 调用 `PullFromAgent()`。
- **"拉取完成 ✓"** — 绿色成功提示。
- 冲突处理同 [冲突对话框](#101-冲突对话框)。

## 5. 仓库收藏

直接浏览并导入已关注 Git 仓库中的 Skills，无需先导入到库中。

- 仓库扫描会递归遍历整个克隆目录，因此 `plugins/<plugin>/skills/<name>` 这类嵌套 Skill 目录也会被发现；`skill.md` 文件名按大小写不敏感匹配。

### 仓库同步 与 已安装 Skill 更新的区别

- 仓库卡片上的 **更新** 和工具栏的 **全部更新**，更新的是该收藏仓库在本地的缓存克隆。
- 这样会让 **仓库收藏** 页面看到最新仓库内容，便于继续浏览或导入新的 Skill 文件。
- 如果云端恢复把新的收藏仓库同步到了当前设备，SkillFlow 会立即在本地克隆这些新恢复下来的仓库，这样搜索、导入和直接推送到智能体都不需要再手动点一次更新。
- 它 **不会** 覆盖 **我的skills** 中已经安装好的那一份 Skill。
- 如果某个 Skill 已经导入到库中，要更新那份已安装副本，仍然要回到 **我的skills（主面板）** 使用卡片上的 **更新**。

        v                                           v

## 6. 云端备份

将 Skill 库镜像备份至云端。支持两种后端：**对象存储**（阿里云 OSS、AWS S3、Azure Blob Storage、Google Cloud Storage、腾讯云 COS、华为云 OBS）和 **Git 仓库**。

### 状态提示

- **云备份未启用提示**（黄色横幅） — 未配置时显示，引导前往设置。

### 操作

| 按钮 | 对象存储标签 | Git 标签 |
|------|------------|---------|
| **立即备份**（Upload 图标） | 立即备份 | 立即备份 |
| **恢复 / 拉取**（Download 图标） | 从云端恢复 | 拉取远端 |
| **刷新**（RefreshCw） | 重新加载最近一次备份的变更结果 | 同左 |

- 云备份未配置时，两个操作按钮均禁用。
- **"备份完成 / Git 同步完成"**（绿色）/ **"备份/同步失败"**（红色）状态消息。

### 本次备份变更列表

- 每次备份或恢复成功后，页面只显示本次操作涉及的文件，不再展示远端全量文件集合。
- 备份页会显示当前应用会话中最近一次成功备份/恢复的**上次同步完成时间**。
- 对象存储与 Git 共用同一种结果列表，按 `新增`、`修改`、`删除` 三种动作展示。
- 已删除文件显示删除标记，不显示文件大小。
- 点击刷新会重新读取当前应用会话里最近一次备份结果。
- 可滚动容器（最大高度限制）。
- **统一备份范围（所有服务商）** — 备份始终以 `AppDataDir` 作为 Git/cloud 工作根目录。会参与同步的内容固定位于其中的 `skills/`、`meta/`、`prompts/` 等目录；本地专属的 `cache/`、`runtime/`、`logs/`、`meta_local/`、`.git/`、`config_local.json`、`star_repos_local.json` 会被排除。收藏仓库的 clone cache 位于 `repoCacheDir`，也不会进入备份范围。
- **对象存储自定义前缀** — 对象存储服务商允许用户填写父级 `remotePath`；SkillFlow 最终总是写入 `<存储桶>/<remotePath>/skillflow/`（若父级路径为空，则为 `<存储桶>/skillflow/`）。
- **云服务商独立配置档案** — 每个云服务商都会保留自己独立的存储桶、路径和凭据配置；在设置中切换服务商时，会恢复该服务商上次保存的表单值，而不会覆盖其他服务商的配置。
- **同步路径可移植** — `meta/*.json` 等同步元数据中的本地路径会以 `AppDataDir` 下的正斜杠相对路径保存，确保 macOS 与 Windows 间恢复后仍能正确定位。
- **Skill 易变检查时间仅本地保存** — 变化频繁的 `LastCheckedAt` 会写入本地 `meta_local/*.local.json`，不参与 git/云端同步，降低多机并行时的冲突概率。
- **收藏仓库同步运行态仅本地保存** — 每个仓库的 `lastSync` 与 `syncError` 会写入本地 `star_repos_local.json`，避免某台设备后台同步导致其他设备的同步元数据频繁抖动与冲突。
- **本地路径配置隔离** — `config_local.json` 保存机器相关的文件系统路径与代理设置，例如 `repoCacheDir`、智能体 `ScanDirs` / `PushDir` / `MemoryPath` / `RulesDir` 以及代理配置；该文件不参与备份和 git 同步。
- **其他设备本地状态仅本地保存** — 自动推送目标智能体、开机自启注册状态、最近一次窗口尺寸等设备级状态会继续保存在 `config_local.json` 中，因此某一台机器执行恢复时不会覆盖另一台机器的本地使用习惯。
- **云端敏感凭据仅本地保存** — Access Key ID、Secret Key、访问令牌等敏感云凭据只会写入 `config_local.json` 中按服务商分组的本地配置；可同步的 `config.json` 仅保留云服务商、存储桶、远端路径、Endpoint、仓库地址、分支等非敏感配置。
- **Git 备份兼容处理** — 当 Git 备份以父目录作为工作树时，SkillFlow 会自动迁移旧的 `skills/.git` 嵌套元数据，避免真实 Skill 文件被当作嵌套仓库而无法跟踪。
- **恢复后的本机补偿动作** — 云端恢复成功后，当前设备上新恢复出来的库内 Skill 会自动推送到已勾选的自动推送目标智能体；新恢复出来的收藏仓库也会立刻在本地完成克隆。

### 服务商覆盖范围

- 对象存储除阿里云 OSS、腾讯云 COS、华为云 OBS 外，还支持 AWS S3（bucket + region）、Azure Blob Storage（container + account name + 可选 service URL）以及 Google Cloud Storage（bucket + service-account JSON 或本地 key file path）。
- 可同步的连接字段现在除既有的 endpoint / repo URL / branch 外，还包括 `region`、`account_name`、`service_url`。Account key、service-account JSON 等敏感值仍只保存在本地 `config_local.json`。

### 自动备份

以下操作完成后（在云备份已启用时）自动触发：

- 删除 Skill / 批量删除
- 新建 / 编辑 / 删除提示词
- 手动导入
- 从智能体拉取
- 更新 Skill
- 从收藏仓库导入

通过 Wails 事件系统向前端推送进度事件：`backup.started`、`backup.progress`、`backup.completed`、`backup.failed`。

### Git 同步（仅 git 服务商）

选择 **git** 服务商后：

- **仓库自举初始化** — 若 Skills 目录还不是 Git 仓库，SkillFlow 会自动执行初始化，并按配置仓库地址设置 `origin`。
- **远端绑定自动修复** — 若 `origin` 缺失或地址变更，拉取/推送前会自动补齐或更新。
- **启动时自动拉取** — 每次 App 启动时，自动对 Git 备份根目录执行 `git pull`，拉取远端最新内容。
- **启动后台任务错峰** — 启动拉取依然会自动执行，但不会再和其他所有启动检查在同一时间爆发；SkillFlow 会在界面可交互后分批启动这些后台任务。
- **远端分支缺失容错** — 首次配置时若远端分支尚未创建，启动拉取会跳过而非直接报错。
- **变更后自动推送** — 与对象存储相同的触发时机；执行 `git add -A && git commit && git push`。
- **排除路径自动清理** — 每次 Git 备份推送前，像 `cache/`、`runtime/` 这样的本地专属排除目录，如果曾被旧版本错误跟踪，会自动从 Git 索引中移除。
- **定时自动同步** — 由"定时自动同步间隔"设置控制（单位：分钟；0 表示禁用）。后台定时器按配置间隔调用 `autoBackup()`。
- **手动操作冲突检测** — 点击 **立即备份** 与 **拉取远端** 时都会检测 Git 冲突/分叉，若需要用户决策会触发 `git.conflict` 事件。
- **冲突解决对话框** — 若 `git pull` 或 `git push` 检测到冲突或历史分叉，弹出模态框：
  - 对话框会在可用时展示冲突相关文件列表。
  - **"以本地为准"** — 中止合并，将本地状态强制推送到远端。调用 `ResolveGitConflict(true)`。
  - **"以远端为准"** — 中止合并，将本地重置为 `origin/<分支>`。调用 `ResolveGitConflict(false)`。
  - **"手动解决"** — 在系统文件管理器中打开 Git 备份根目录，方便用户直接检查并处理冲突文件。
  - “以本地为准”和“以远端为准”成功后都会从磁盘重载应用状态（skills/meta/config），并发送 `git.sync.completed` 事件。
- **拉取后即时刷新** — 启动自动拉取或手动拉取成功后，会立刻重载磁盘状态，使 `meta/` 与配置变更立即生效。
- 若冲突发生在启动阶段（前端尚未加载），将以待处理标志存储，并在备份页面挂载时通过 `GetGitConflictPending()` 弹出对话框。

### 云存储标签页

| 控件 | 说明 |
|------|------|
| **云服务商按钮** | 以可自动换行的卡片网格展示。**Git 仓库** 会始终固定在第一位，其余服务商保持后端返回顺序；切换时会自动恢复该服务商各自保存的存储桶 / 路径 / 凭据草稿 |
| **存储桶名称** | 对象存储桶名称（选择 git 服务商时隐藏） |
| **远程路径** | 对象存储的父级路径（可选）。用户只需填写父目录，SkillFlow 会自动在末尾追加 `/skillflow/` 作为最终备份前缀 |
| **最终备份路径预览** | 实时渲染对象存储的最终目标路径，显示为 `<存储桶>/<remotePath>/skillflow/`，方便用户在保存前确认真实备份位置 |
| **凭据字段** | 根据 `RequiredCredentials()` 动态渲染 — 文本或密码输入框，因服务商而异。Endpoint / 仓库地址 / 分支等非敏感连接信息保存在 `config.json`，Access Key / Secret / Token 等敏感凭据仅保存在 `config_local.json`。Git 字段：仓库地址、分支、用户名、访问令牌 |
| **输入规范化** | 阿里云 OSS 和华为云 OBS 的存储桶字段既可填写纯桶名，也可填写常见的完整桶域名/URL，保存时会自动提取桶名。腾讯云 COS 与其他对象存储一样使用独立的存储桶 + Endpoint 模型。存储桶始终来自单独的存储桶输入框，而 Endpoint 字段既可填写纯 Endpoint host，也可填写完整桶域名/URL，并会按用户输入形式保留用于展示 |
| **附加服务商细节** | AWS S3 在保存前会去除 region 字段首尾空白。Azure Blob Storage 使用存储桶字段作为 container 名称，将 account name 和可选 service URL 作为可同步字段保存，并在 service URL 为空时默认使用 `https://<account>.blob.core.windows.net/`。Google Cloud Storage 支持填写内联 service-account JSON 或本地 key file path，并且该凭据始终只保存在本地 `config_local.json`。 |
| **定时自动同步间隔** | 数字输入框（分钟）；0 表示仅在变更后同步；正数启动后台定时器 |
| **启用自动云备份开关** | 控制每次写操作后是否自动触发备份，同时控制定时器 |

### 通用标签页

|------|------|

| **打开日志目录** | 一键在系统文件管理器中打开本地日志目录；若目标不存在，则回退到最近存在的父目录 |

- 最多仅保留 **2 个文件**：`skillflow.log` 和 `skillflow.log.1`。
- 单文件最大 **1MB**。
- 当 `skillflow.log` 达到上限后会滚动，并覆盖较旧的备份日志文件。

| 元素 | 说明 |

| **状态条** | 同时显示来源徽章与当前页面允许展示的状态（例如“已导入 / 可更新 / 已推送智能体”），并保持在同一块紧凑头部区域内；像“已导入”和“已推送智能体”这类并存状态在放得下时会优先同排展示，放不下时再自动换到下一行，确保完整可见 |

| **打开目录按钮**（悬停） | 同主面板卡片 |
| **选择复选框**（右下角） | `showSelection = true` 时显示 |

### 统一状态语义

| 状态 | 含义 |
|------|------|
| **installed** | 该逻辑 skill 已经在 **我的skills** 中存在至少一个已安装实例 |
| **imported** | 外部来源页面对 **installed** 的文案别名；在 GitHub / 仓库收藏 / 智能体页面中表示“已经在我的skills里” |
| **pushed** | 该逻辑 skill 已经存在于某个智能体配置的 **PushDir** 中 |
| **pushedAgents** | 哪些智能体的 **PushDir** 当前包含该逻辑 skill；卡片用它来渲染智能体图标列表 |
| **seenInAgentScan** | 该逻辑 skill 已在某个智能体配置的 **ScanDirs** 中被扫描到；表示智能体里某处已经有它，但不一定是 SkillFlow 推过去的。当前它主要用于分组和关联，不再作为卡片 badge 单独展示。 |
| **updatable** | 某个已安装的 Git Skill，其远端最新 SHA 比本地 `SourceSHA` 更新 |

这些状态由后端基于统一逻辑主键模型计算；前端页面不再独立根据 `Name` 或 `Path` 推断。

```text
                 统一状态语义示意图

[GitHub 候选项] [仓库收藏 Skill] [智能体扫描候选项]
        \              |                /
         \             |               /
          +------ 逻辑上是同一个 skill ------+
                         |
                         v
                  [我的skills 实例]
                   installed = true
             外部页面显示 imported = true
                         |
                         +---- 复制到智能体 PushDir --> pushed = true
                         |
                         +---- 远端有更高 SHA -------> updatable = true

[智能体 ScanDirs] --- 扫描到同一逻辑 skill ---> seenInAgentScan = true
```

## 11. 后端事件

Go 后端通过 Wails runtime 向前端推送的事件：

| 事件 | 触发时机 | 载荷 |
|------|---------|------|
| `backup.started` | 自动备份开始 | — |
| `backup.progress` | 每上传一个文件 | `{ currentFile: string }` |
| `backup.completed` | 备份完成 | `{ files: Array<{ path, size, action }> }` |
| `backup.failed` | 备份出错 | — |
| `update.available` | 发现 Skill 的缓存仓库中有新提交 | `{ skillID, skillName, currentSHA, latestSHA }` |
| `star.sync.progress` | 单个仓库同步完成 | `{ repoURL, repoName, syncError }` |
| `star.sync.done` | 所有仓库同步完成 | — |
| `git.sync.started` | Git 拉取/推送开始 | — |
| `git.sync.completed` | Git 同步成功 | 由备份 / 恢复 / 冲突解决触发时为 `{ files: Array<{ path, size, action }> }` |
| `git.sync.failed` | Git 同步出错 | — |
| `git.conflict` | 检测到 Git 合并冲突 | `{ message: string, files?: string[] }` |

主面板监听 `update.available` 事件，实时在对应 Skill 卡片上标记红色更新红点。
备份页监听所有 `git.*` 事件，并在收到 `git.conflict` 时弹出冲突解决对话框。
`App.tsx` 监听全部三个 `app.update.*` 事件，解析发出的 `AppUpdateInfo` payload，并驱动更新对话框的状态机。

## 12. 应用更新对话框

检测到新版本时弹出的居中模态对话框，同时适用于启动时自动检测和设置页手动检测两个入口。对话框由四态状态机驱动：

| 状态 | 触发条件 | 对话框内容 |
|------|---------|----------|
| `available` | 收到 `app.update.available` 事件 | 版本号 + 更新说明 + 平台对应操作按钮 |
| `downloading` | 用户点击"下载并自动重启更新"（仅 Windows） | 旋转图标 + 下载进度提示 |
| `ready_to_restart` | 收到 `app.update.download.done` 事件 | 完成提示 + "立即重启" / "稍后重启"按钮 |
| `download_failed` | 收到 `app.update.download.fail` 事件 | 错误提示 + "前往下载页"按钮 |

### 平台行为

启动检测与手动检测均使用同一模态对话框。

- **Windows** — `available` 状态提供三个选项：
  1. **下载并自动重启更新** — 后台下载新 exe，完成后提示重启。
  2. **前往 Release 页面手动下载** — 在系统浏览器打开 GitHub Releases 页面。
  3. **跳过此版本（下次启动不再提示）** — 记住跳过的版本；启动检测不再对该版本弹窗，手动检测不受影响，仍正常弹出。
- **macOS** — `available` 状态提供两个选项（暂不支持自动下载）：
  1. **前往 Release 页面手动下载** — 在系统浏览器打开 GitHub Releases 页面。
  2. **跳过此版本（下次启动不再提示）** — 与 Windows 行为一致。
- 两个平台都使用同一份 `AppUpdateInfo` payload 渲染当前版本、最新版本和 Release 页面入口。

### 跳过版本机制

- 跳过的版本 tag 存储在 `AppConfig.SkippedUpdateVersion`，并写入共享的 `config.json`，因此重启应用后仍会生效。
- 启动检测时，若 `latestVersion == skippedUpdateVersion`，则**不**发布 `app.update.available` 事件，不弹出对话框。
- 设置页手动点击"检测更新"时，`CheckAppUpdateAndNotify` 始终发布事件，不受跳过版本影响 — 手动检测总是正常弹出对话框。
- 点击"跳过此版本"时调用 `SetSkippedUpdateVersion(latestVersion)`。

### 手动检测按钮（设置页）

设置页标题栏右上角提供 **"检测更新"** 按钮：

- 按钮旁显示当前应用版本号（`vX.Y.Z`）。
- 点击 → 调用 `CheckAppUpdateAndNotify()`；检测中按钮显示旋转图标。
- 若发现新版本，更新对话框通过 `app.update.available` 事件自动弹出。
- 若已是最新版本，内联显示"已是最新版本 (vX.Y.Z)"。
- 出错时内联显示"检测失败: …"。

### 操作说明

| 控件 | 操作 |
|------|------|
| **下载并自动重启更新**（Windows，`available` 状态） | `DownloadAppUpdate(downloadUrl)` — 启动后台异步下载 |
| **前往 Release 页面手动下载**（`available` 状态） | `OpenURL(releaseUrl)` — 在浏览器打开发布页；关闭对话框 |
| **跳过此版本**（`available` 状态） | `SetSkippedUpdateVersion(latestVersion)` — 持久化跳过版本；关闭对话框 |
| **立即重启**（`ready_to_restart` 状态） | `ApplyAppUpdate()` — 写入批处理脚本并退出；脚本完成 exe 替换后重启 |
| **稍后重启**（`ready_to_restart` 状态） | 关闭对话框，不立即重启 |
| **前往下载页**（`download_failed` 状态） | `OpenURL(releaseUrl)` — 在浏览器打开发布页；关闭对话框 |
| **×**（除 `downloading` 外所有状态） | 关闭对话框（本次会话内不再提示） |

### 后端方法

| 方法 | 说明 |
|------|------|
| `GetAppVersion()` | 返回当前版本字符串（CI 通过 `-ldflags` 注入；本地开发为 `"dev"`） |
| `CheckAppUpdate()` | 查询 GitHub Releases API；返回含平台匹配下载链接的 `AppUpdateInfo` |
| `CheckAppUpdateAndNotify()` | 调用 `CheckAppUpdate()` 并在有更新时发布 `EventAppUpdateAvailable`；始终通知，不受跳过版本影响 |
| `GetSkippedUpdateVersion()` | 返回 config 中存储的用户跳过版本 tag |
| `SetSkippedUpdateVersion(version)` | 将跳过版本 tag 持久化到 config；传空字符串可清除 |
| `DownloadAppUpdate(url)` | 异步下载新 exe 到临时目录；完成后发布 `app.update.download.done` 或 `app.update.download.fail` 事件 |
| `ApplyAppUpdate()` | 仅 Windows — 写入批处理脚本实现退出后 exe 替换，然后调用 `os.Exit(0)` |

### 版本注入（CI）

GitHub Actions 在编译时通过 ldflags 注入 tag 名：
```
wails build -ldflags "-X main.Version=${{ github.ref_name }}"
```
当 `Version == "dev"` 时（本地开发），启动时的更新检测会跳过。

## 13. 我的智能体

浏览每个已启用智能体当前目录中的 Skills。

### 布局

### 工具栏

| 控件 | 说明 |
|------|------|
| **Skills \| Memory** | 在 Skill 管理视图和记忆预览视图之间切换右侧主内容 |

### Memory 面板

- 仅在顶部切换到 **Memory** 时显示。
- 直接读取当前选中智能体配置好的主记忆文件和规则目录，展示该智能体当前磁盘上的真实记忆内容。
- 分区头部提供 **刷新** 操作，可在不离开 **我的智能体** 的情况下重新读取该智能体当前落地的记忆状态。
- **记忆文件** 卡片展示配置路径、**打开文件** 操作以及当前文件内容预览；如果路径未配置，或已配置但文件暂时不存在，会保留路径区域并显示对应空状态，而不是让整个页面报错。
- **规则目录** 区域展示配置路径、**打开目录** 操作，以及该目录下每个平铺 `.md` 文件的预览卡片。
- 以 `sf-` 开头的规则文件会标记为 **托管**，方便区分 SkillFlow 推送的模块记忆和其他智能体本地规则文件。
- 顶部共享搜索框只会在 **Memory** 激活时过滤记忆预览；当搜索后没有匹配项时，当前面板会显示专门的空搜索提示，而不是复用 Skill 分区的空状态。

### Skills 面板

- 仅在顶部切换到 **Skills** 时显示。

### 推送路径分区

- 显示当前智能体推送目录中的可删除 Skill。
- 推送路径扫描会复用 **设置 → 通用** 中相同的深度限制（默认 `5`，保存范围 `1-20`）。
- 多选模式下，**全选 / 取消全选** 仅作用于当前可见、已过滤后的推送路径卡片。
- 如果智能体目录中的 Skill 能关联到“我的skills”里已安装的记录，卡片会额外显示其来源徽标，区分手动导入和 Git 来源导入。
- 卡片继续显示“已导入 / 可更新 / 已推送到其他智能体”等真正有决策价值的状态；其中 `pushed` 使用紧凑的智能体图标列表，并会自动排除当前正在查看的智能体。

### 扫描路径分区

- 显示仅通过扫描目录发现的只读 Skill。
- 扫描路径发现会复用 **设置 → 通用** 中相同的深度限制（默认 `5`，保存范围 `1-20`）。
- 与推送路径共享同一组面板级搜索和排序控件。
- 扫描路径卡片也使用同一套紧凑状态条；当扫描项能关联到“我的skills”里已安装的记录时，还会显示来源 / 已导入 / 可更新 / 已推送到其他智能体等状态。“被扫描到”这件事则由分区本身表达，不再在每张卡片上重复显示 `detected` 徽章。

## 14. 我的提示词

用于在同步的 `prompts/` 目录中保存可复用的 system prompt。

### 导航与存储

- 左侧边栏在 **我的智能体** 下方新增 **我的提示词** 入口。
- 每条提示词都会在备份根目录下保存 `prompts/<category>/<name>/system.md`，并配套保存 sidecar 元数据文件 `prompts/<category>/<name>/prompt.json`，因此对象存储服务商与 Git 服务商都会自动同步同一批提示词文件。
- 提示词名称为必填项，在整个提示词库内全局唯一，并作为文件夹键名使用。

### 页面布局

- 页面改为和 **我的skills** 一致的左侧分类栏 + 右侧内容区结构。
- 内置 **Default** 作为默认分类。
- 与其他主路由一致，重新进入 **我的提示词** 页面时会重新挂载页面并拉取当前提示词数据，而不是长期保留旧页面实例。
- 提示词卡片会同时受到分类筛选、关键字搜索和 A-Z / Z-A 排序控制。
- 搜索支持逻辑 `and` / `or` 语法，例如 `review and golang` 或 `summary or changelog`。

### 分类管理

- 可在左侧边栏新建分类。
- 非默认分类支持右键重命名与删除。
- 提示词卡片支持拖拽到分类上移动，交互与 **我的skills** 的分类拖拽一致。

### 提示词卡片

- 卡片复用 Skill 卡片的视觉语言（`card-base`、悬停发光、紧凑操作区）。
- 卡片默认展示 **名称**、可选 **描述** 和内容开头摘要。
- 提示词卡片列表中不再展示关联图片缩略图和网页链接胶囊，让列表更聚焦于文本摘要。
- 当内容超出预览范围时，会显示 **点击查看更多** 提示。
- 右上角按钮支持一键复制完整提示词内容到桌面剪贴板，在 Windows 上也会保留多行内容。
- 提示词复制现在会依次回退桌面运行时剪贴板、浏览器剪贴板和文档复制 API，因此某一条复制链路不可用时仍能完成复制。
- 右下角 **删除** 操作会先弹出确认对话框，再删除卡片和对应的提示词文件夹。

### 提示词编辑器

- 点击 **添加提示词** 会打开内置编辑器，包含 **名称**（必填）、**描述**（可选）、**分类** 和完整 `system.md` 内容。
- 编辑器支持填写最多 **3 个关联图片 URL**。已添加的图片会保留在图片区展示，而图片输入条会移动到正文之后的统一附件输入区，减少阅读干扰。
- 每张已添加图片的右上角都会提供删除操作。
- 点击编辑器里的图片缩略图，会在当前编辑器上方打开应用内放大预览层，而不是跳转到外部浏览器。
- 编辑器现在会限制弹窗正文和提示词正文输入区的高度，在窗口较小时改为内部滚动，避免底部确认按钮被挤出 SkillFlow 窗口。
- 编辑器支持通过放在正文之后统一附件输入区里的单行 markdown 输入框添加 **网页链接**。点击 **添加** 后，会把 `[产品文档](https://example.com/doc)` 中中括号里的文字作为可点击链接标签显示在上方，并清空当前输入框。
- 点击已有提示词卡片会打开同一编辑器，并载入现有内容以便编辑和重命名。
- 保存后会把正文写回 `prompts/<category>/<name>/system.md`，并把描述、图片 URL、网页链接等元数据写入 `prompts/<category>/<name>/prompt.json`。

### 导入 / 导出

- 工具栏 **导入** 支持读取 JSON 提示词库文件，并保留每条提示词在导入文件里的分类；如果本地已存在同名提示词，会在真正写入前让用户选择 **跳过** 还是 **覆盖**。
- 导入冲突对话框支持勾选 **对剩余 {count} 个冲突执行相同操作**，可把当前决策复用到本次导入剩余冲突，但不会保存成全局偏好。
- 工具栏 **导出** 不再点击后立刻落盘，而是先在页面内展开一条导出栏。
- 导出栏支持 **全部**、左侧当前选中分类，以及 **指定** 三种范围。
- **指定** 支持在左侧当前范围内多选提示词导出；左侧选中 **全部** 时可在全库多选，选中具体分类时只会在该分类内多选。
- 导出的 JSON 仍会保留每条提示词自己的分类、图片 URL 和网页链接。

## 15. 我的记忆

SkillFlow 提供统一的记忆管理界面，支持在一处编写和管理个人 AI 编程助手记忆，并将其推送到多个 AI 工具。

### 记忆类型

- **主记忆**：单个 `main.md` 文件，包含跨所有已配置智能体共享的全局指令。
- **模块记忆**：按主题划分的独立 Markdown 文件（如 `coding-style`、`testing-rules`），存放于 `rules/` 目录下。

### 页面布局

「我的记忆」页面展示：
- 带有各智能体推送状态标签的**主记忆卡片**（置顶）。
- 支持搜索、智能体筛选和排序的**双列模块网格**。
- 右侧**编辑抽屉**（约占页面宽度 55%），含「编辑 / 预览」选项卡、推送目标选择、推送模式、自动推送开关及「立即推送」按钮。

### 推送模式（按智能体）

| 模式 | 行为 |
|------|------|
| **合并** | SkillFlow 在智能体记忆文件内写入托管标记块，块外内容保留不变。 |
| **覆盖** | SkillFlow 完全掌控智能体的记忆文件和规则目录。 |

### 推送状态

每个智能体显示以下三种状态之一：

| 状态 | 含义 |
|------|------|
| ✓ 已同步 | 最近一次推送与当前本地记忆一致。 |
| ⚠ 待推送 | 自上次推送以来本地记忆已发生变更。 |
| 从未推送 | 已配置推送目标但尚未推送。 |

当任一目标智能体有待推送内容时，侧边栏入口和对应卡片上将显示红点。

### 模块文件命名

模块记忆写入智能体规则目录时会添加 `sf-` 前缀（如 `coding-style` → `sf-coding-style.md`）。

### 外部编辑器

每张记忆卡片提供「在编辑器中打开」按钮，可用系统默认文本编辑器打开文件。SkillFlow 会检测文件变更并自动刷新内容。

### 智能体设置

在**设置 → 智能体**中，每个智能体新增以下配置项：
- **记忆文件**：智能体主记忆文件路径（显示默认值，用户可修改）。
- **规则目录**：智能体规则目录路径。
- **记忆推送模式**：合并 / 覆盖。
- **自动推送记忆**：记忆内容变更时自动推送。

### 云备份

记忆内容文件（`memory/main.md` 和 `memory/rules/*.md`）已纳入云备份范围。本地配置文件（`memory/memory_local.json`）不参与备份。
