## 云备份新增服务商

- 对象存储在原有阿里云 OSS、腾讯云 COS、华为云 OBS 之外，新增 AWS S3、Azure Blob Storage、Google Cloud Storage。
- AWS S3 使用 `bucket + region`。
- Azure Blob Storage 使用 `container`（即设置页里的 bucket 字段）+ `account name` + `account key`，并支持可选 `service URL`。
- Google Cloud Storage 使用 `bucket + Service Account JSON`，该凭据也可以填写为本地密钥文件路径。
- `region`、`account_name`、`service_url` 会作为可同步连接字段保存在 `config.json`；`account key` 和 `Service Account JSON` 等敏感凭据仍只保存在 `config_local.json`。

## 1. 导航与外壳

固定左侧边栏（宽 w-56）提供全局导航。

| 路由 | 图标 | 标签 |
|------|------|------|
| `/` | Package | 我的skills |
| `/tools` | Wrench | 我的工具 |
| `/prompts` | FileText | 我的提示词 |
| `/sync/push` | ArrowUpFromLine | 推送到工具 |
| `/sync/pull` | ArrowDownToLine | 从工具拉取 |
| `/starred` | Star | 仓库收藏 |
| `/backup` | Cloud | 云备份 |
| `/settings` | Settings | 设置 |

- 当前路由：使用当前主题色的浅色块高亮，并配合克制的描边与层次阴影。
- 非当前路由：灰色文字，鼠标悬停时显示高亮。
- 侧边栏左上角：`SkillFlow` 标题文字左侧显示应用图标；图标会略高于标题文字以保证辨识度，并保持原始长宽比。
- 侧边栏右上角：**Languages** 语言快捷按钮，可在 **中文** 与 **English** 之间即时切换，并将偏好持久化到 `localStorage`。
- 侧边栏右上角：**调色盘** 主题快捷按钮，按 **Dark → Young → Light** 顺序即时轮换。
- 左下角 **意见反馈** 按钮：在默认浏览器中打开 GitHub 新建 Issue 页面。
- 窗口关闭行为：点击左上角关闭按钮时仅隐藏主窗口，应用继续在后台运行。
- 窗口启动尺寸：启动时会优先恢复保存在本地 `config_local.json` 里的最近一次窗口尺寸；如果用户还没有手动调整过，则会根据当前显示器尺寸自适应计算初始窗口，使用更适合桌面的默认大小，并在屏幕可用空间内居中显示。
- macOS 托盘行为：应用启动后会在顶部状态栏创建黑白状态栏图标；主窗口隐藏后会同时移除 Dock 图标，仅保留该状态栏图标。单击状态栏菜单可执行 `Show SkillFlow`、`Hide SkillFlow` 和 `Quit SkillFlow`。
- Windows 托盘行为：应用驻留在系统托管区，并显示应用自身图标；点击托盘图标会弹出菜单，支持 `Show SkillFlow` 和 `Exit`。

## 2. 我的skills（主面板）

集中管理你的 Skill 库。

### 工具栏

| 控件 | 操作 |
|------|------|
| **搜索框** | 加宽显示，实时按技能名过滤（不区分大小写）；窗口较窄时工具栏会自动换行，保证按钮完整可见 |
| **排序切换** | 两个按钮在 **A-Z** 与 **Z-A** 间切换，按 Skill 名称首字母正序或逆序排列 |
| **更新**（RefreshCw） | 调用后端 `CheckUpdates()`；按规范化后的仓库来源 + 子路径聚合同一个 Git Skill，刷新 `LatestSHA` / `LastCheckedAt`，为可更新卡片显示红点与“更新”操作，并在实例已是最新时清理过期更新标记。该按钮只检查，不会直接覆盖本地文件 |
| **批量删除**（CheckSquare） | 切换多选模式 |
| **导入**（FolderOpen） | 打开系统文件夹选择器 → `ImportLocal(dir)` |
| **远程安装**（Github） | 打开 GitHub 安装对话框 |

### 自动推送目标工具

- 工具栏下方新增一条专用选择区，可用与 **推送到工具** 页面一致的图标按钮样式，勾选一个或多个已启用工具作为**自动推送目标**。
- 这组选择只保存在当前设备本地，并会在后续继续复用。
- 任何新添加到 **我的skills** 的 Skill，在成功导入库之后都会自动复制到这些选中的工具中；覆盖本地导入、GitHub 安装、从工具拉取、以及仓库收藏导入这几条入口。
- 自动推送是非破坏性的：如果目标工具的 `PushDir` 中已经存在同名 Skill，SkillFlow 会跳过该目标，不会覆盖已有内容。
- 之后取消勾选某个工具时，也不会删除之前已经推送过去的 Skill；如需移除，仍需用户在 **我的工具** 页面或工具目录里手动删除。

### 多选模式（点击"批量删除"激活）

| 控件 | 操作 |
|------|------|
| **全选 / 取消全选** | 切换当前过滤后所有 Skill 的选中状态 |
| **删除（n 个）**（Trash2，红色） | `DeleteSkills(ids)` — 未选择时置灰禁用 |
| **取消** | 退出多选模式并清空选择 |

### 分类侧边栏

- 列出所有分类；点击可过滤 Skill 网格。
- **"全部" 按钮** — 显示所有分类的 Skills。
- **拖拽目标区域** — 将 Skill 卡片拖到某个分类，即可移动至该分类。
- **拖拽高亮** — 拖动 Skill 卡片悬停到分类上方时，目标分类会以更明显的高亮状态显示。
- **右键上下文菜单**（每个分类）：
  - **重命名** — 显示内联文字输入框；回车确认，Esc 取消；调用 `RenameCategory()`。（`Default` 分类不可用）
  - **删除** — 空分类会直接调用 `DeleteCategory()` 删除；若该分类下仍有 Skill，则弹出提示框要求先清空该分类后再删除。（`Default` 分类仍不可删除）
- **新建分类**（底部 Plus 图标） — 显示内联文字输入框；回车或失焦确认，Esc 取消；调用 `CreateCategory()`。

### Skill 网格

- 布局：3 列，宽屏 4 列。
- **空状态** — 显示"没有找到 Skills"及使用提示。
- **Skill 右键菜单** — 包含"移动到其他分类"项（仅显示非当前分类）、删除，以及适用时的更新操作。
- **拖拽操作** — 将 Skill 卡片拖至侧边栏分类可移动；开始拖拽后，会有一个缩小版卡片跟随鼠标移动；命中左侧分类后，原卡片会收缩成一条细线，直到结束拖拽；将文件夹从操作系统文件管理器拖入窗口可直接导入。
- **窗口级拖拽遮罩** — 拖拽文件进入窗口时，显示半透明靛蓝色遮罩和"松开以导入 Skill"提示。
- **悬浮提示** — 鼠标悬停卡片 300 毫秒后出现（详见 [Skill 悬浮提示](#9-skill-悬浮提示)）。

### Skill 更新流转

- 工具栏的 **更新** 只负责对已安装的 Git Skill 做远端版本检查，并把卡片标记为“可更新”。
- 卡片上的 **更新** 才会真正下载最新文件并覆盖 **我的skills** 中已安装的那一份副本。
- 远端检查会按应用内统一使用的 Git 逻辑主键聚合：规范化仓库来源 + 仓库子路径。
- 当多个已安装实例指向同一个逻辑 Git Skill 时，远端 SHA 只查询一次，但每个实例仍会用自己的 `SourceSHA` 决定自己是否 `updatable`。

```text
[主面板工具栏 更新]
         |
         v
    CheckUpdates()
         |
         v
比较本地 SourceSHA 与远端最新 SHA
         |
         +--> 没有更新  -> 卡片保持不变
         |
         +--> 有更新    -> 写入 LatestSHA -> 显示红点 / 更新按钮

[Skill 卡片 更新]
         |
         v
 UpdateSkill(skillID)
         |
         v
重新下载该 skill 对应的仓库子路径
         |
         v
覆盖“我的skills”中的安装目录
         |
         v
SourceSHA = LatestSHA -> 清除更新标记
```

## 3. 推送到工具

将 Skill 库中的 Skills 复制到外部工具目录。

### 布局

- 采用与"我的skills"接近的双栏布局。
- 左侧边栏显示分类筛选：**全部** 加所有现有分类。
- 右侧显示目标工具选择、搜索 + A-Z/Z-A 排序控件、推送模式按钮，以及当前分类范围内的 Skill 卡片网格。

- 页面间距与网格密度做了收紧，配合启动时自适应的窗口尺寸，在常见笔记本和桌面分辨率下，通常可以在首屏同时看到头部控件、Skill 列表和底部推送操作，而不是一打开就立刻需要滚动。
### 目标工具选择

- 每个已启用的工具对应一个切换按钮（图标 + 名称）。
- 可同时选择多个工具。
- 当前激活的分类、工具和范围按钮会使用更亮的主题底色、更浅的边框和更明显的发光，因此在深色主题下不需要额外符号也能清楚区分选中状态。

### 同步范围

推送行为由左侧分类筛选和右侧模式按钮共同决定：

| 模式 | 行为 |
|------|------|
| **手动选择** | 位于 **推送全部 / 推送当前分类** 左侧，且默认选中；以当前左侧分类范围作为候选列表，在卡片上逐个勾选，也可对当前列表执行全选 |
| **推送全部 / 推送当前分类** | 左侧选中 **全部** 时推送整个库；选中具体分类时只推送该分类 |

### 目录缺失检查

推送前，应用调用 `CheckMissingPushDirs()`。若目标工具目录尚未创建，弹出确认对话框：

- 列出每个缺失的工具名称及完整目录路径。
- **"创建并推送"** — 创建目录后继续推送。
- **"取消"** — 中止，不创建任何目录。

### 冲突处理

若目标目录中已存在同名 Skill，逐条弹出冲突对话框（详见 [冲突对话框](#101-冲突对话框)）。

### Skill 网格

- 这个页面的库内 Skill 卡片只展示推送决策真正相关的状态：哪些工具的 `PushDir` 已经包含该逻辑 skill。
- 已推送工具会用紧凑图标列表表示；数量过多时折叠为省略号，并支持悬浮查看完整列表。

### 底部操作栏

- **"开始推送（n）"** 按钮 — 未选工具或数量为零时禁用；进行中显示"推送中…"。
- **"推送完成 ✓"** — 推送结束后绿色成功提示。

## 4. 从工具拉取

将外部工具目录中的 Skills 导入到你的库中。

### 布局

- 采用与“推送到工具”一致的双栏布局。
- 左侧边栏列出全部分类，并直接作为导入目标分类选择器。
- 右侧显示来源工具选择、扫描反馈、搜索 + A-Z/Z-A 排序控件、可选 Skill 网格和底部执行栏。

### 来源工具选择

- 与推送页相同的切换按钮；切换工具时重置已扫描列表。
- 当前导入目标分类与来源工具沿用推送页相同的高亮底色、浅色边框与发光选中样式，因此在深色主题下也能更稳定地识别当前选择。

### 扫描

- **"扫描"** 按钮 — 调用 `ScanToolSkills(toolName)`，递归搜索该工具已配置的扫描目录中的 `skill.md` 文件。
- 本地工具扫描会复用 **设置 → 通用** 中相同的深度限制（默认 `5`，保存范围 `1-20`）。
- 扫描中显示动效"扫描中…"。
- 扫描失败时显示**红色错误提示**；未发现 Skill 时显示**黄色警告提示**。
- 工具扫描候选项会优先按逻辑主键去重和关联；只有在无法得到稳定主键时才会退回弱匹配，因此同名但内容不同的 Skill 不会再被直接合并。

### Skill 网格

- 扫描成功后显示。
- 搜索框会实时按名称过滤扫描结果。
- 排序切换按钮可在 **A-Z** 与 **Z-A** 间切换。
- 卡片只展示拉取决策真正需要的状态：是否已导入到“我的skills”。
- 每次扫描后，所有 Skill 默认均为未选中。
- 可逐个选择，可对当前可见列表执行 **全选 / 取消全选**，也可通过与“全选”同样的方形切换按钮 **选择未导入**，一键勾选当前可见且尚未导入的 Skill。
- 选择状态和拉取冲突都按扫描到的路径跟踪，因此来自不同工具目录的同名 Skill 也能独立处理。

### 底部操作栏

- **"开始拉取（n）"** 按钮 — 调用 `PullFromTool()`。
- **"拉取完成 ✓"** — 绿色成功提示。
- 冲突处理同 [冲突对话框](#101-冲突对话框)。

## 5. 仓库收藏

直接浏览并导入已关注 Git 仓库中的 Skills，无需先导入到库中。

- 仓库扫描会递归遍历整个克隆目录，因此 `plugins/<plugin>/skills/<name>` 这类嵌套 Skill 目录也会被发现；`skill.md` 文件名按大小写不敏感匹配。
- 仓库递归扫描会受到 **设置 → 通用** 中 **远程仓库递归扫描深度** 控件的限制（默认 `5`，保存范围 `1-20`）。

### 视图模式

| 模式 | 图标 | 描述 |
|------|------|------|
| **文件夹视图** | Folder | 仓库卡片网格；点击卡片进入该仓库的 Skill 列表 |
| **平铺视图** | LayoutGrid | 所有仓库的 Skills 合并显示在一个网格中 |

### 工具栏（普通模式）

| 按钮 | 操作 |
|------|------|
| **搜索框** | 在平铺视图和仓库详情视图中显示；按名称过滤当前 Skill 网格 |
| **排序切换** | 在平铺视图和仓库详情视图中显示；在 **A-Z** 与 **Z-A** 间切换 |
| **批量导入**（CheckSquare） | 仅在有 Skill 网格可见时进入多选模式 |
| **全部更新**（RefreshCw） | `UpdateAllStarredRepos()` — 并行克隆/拉取所有仓库；更新中图标旋转 |
| **添加仓库**（Plus，靛蓝色） | 打开"添加仓库"对话框 |

### 工具栏（多选模式）

| 按钮 | 操作 |
|------|------|
| **全选 / 取消全选** | 切换所有可见 Skills |
| **推送到工具（n）** | 打开推送到工具对话框（见下文） |
| **导入到我的skills（n）** | 打开导入对话框 |
| **取消** | 退出多选模式 |

### 仓库卡片（文件夹视图）

- 点击打开该仓库的 Skill 列表。
- **在浏览器中打开**（ExternalLink 图标） — 在默认浏览器中打开仓库 URL。
- **更新**（RefreshCw 图标） — `UpdateStarredRepo(url)` — 拉取最新提交。
- **删除**（Trash2 图标，悬停变红） — 从收藏列表中移除。
- 卡片下方显示最后同步时间及同步错误信息。

### 内置起始仓库初始化

- 仅在 `star_repos.json` 尚不存在时（设备/配置的首次初始化）才会注入内置起始仓库。
- 当前内置起始仓库：
  - `https://github.com/anthropics/skills.git`
  - `https://github.com/ComposioHQ/awesome-claude-skills.git`
  - `https://github.com/affaan-m/everything-claude-code.git`
- 一旦用户删除内置仓库，持久化列表即为唯一事实来源；后续启动不会再自动把该仓库加回。

### 仓库详情视图（钻取）

- 顶部面包屑返回按钮（ChevronLeft）回到仓库列表。
- Skill 网格与平铺视图的选择/导入操作相同。
- 这个页面的仓库 Skill 卡片只显示“已导入 / 已推送到工具”两类状态。
- “已导入”徽章按规范化仓库来源 + 子路径解析，因此不同仓库里的同名 Skill 不会再互相误判。
- `pushed` 状态会以工具品牌图标显示，并支持悬浮查看完整工具列表。

### 仓库同步 与 已安装 Skill 更新的区别

- 仓库卡片上的 **更新** 和工具栏的 **全部更新**，更新的是该收藏仓库在本地的缓存克隆。
- 这样会让 **仓库收藏** 页面看到最新仓库内容，便于继续浏览或导入新的 Skill 文件。
- 它 **不会** 覆盖 **我的skills** 中已经安装好的那一份 Skill。
- 如果某个 Skill 已经导入到库中，要更新那份已安装副本，仍然要回到 **我的skills（主面板）** 使用卡片上的 **更新**。

```text
                    跨页面更新流转图

[仓库卡片 更新]                           [我的skills 卡片 更新]
        |                                           |
        v                                           v
UpdateStarredRepo(url)                       UpdateSkill(skillID)
        |                                           |
        v                                           v
刷新本地仓库缓存克隆                         覆盖已安装的库内副本
        |                                           |
        v                                           v
刷新仓库收藏中的 Skill 列表                  清除我的skills中的更新标记
        |
        +--> 让后续导入拿到更新后的内容
        x--> 不会直接修改“我的skills”文件
```

### 添加仓库对话框

- URL 输入框（HTTPS 或 SSH 格式）；回车触发添加。
- **"添加"** 按钮 — `AddStarredRepo(url)`。
- 若仓库需要 HTTP 认证，自动弹出 **HTTP 认证对话框**。
- 若 SSH 认证失败，弹出 **SSH 认证错误对话框**，说明所需配置。
- 显示克隆进度状态（"克隆中…"）。

### HTTP 认证对话框

- 用户名 + 密码输入框（密码掩码）；密码框按回车确认。
- **"确认"** — 重试调用 `AddStarredRepoWithCredentials(url, user, pass)`。
- **"取消"** — 中止。
- 凭据错误时显示错误提示。

### SSH 认证错误对话框

- 说明 SSH 密钥配置检查清单：
  - 是否已通过 `ssh-keygen` 生成密钥
  - 公钥是否已添加到 GitHub / GitLab
  - SSH Agent 是否正在运行（`ssh-add`）
  - 建议改用 HTTPS 协议克隆
- **"关闭"** 按钮。

### 导入对话框（到我的skills）

- 分类选择下拉菜单。
- **"导入 n 个"** — `ImportStarSkills(paths, repoURL, category)`。
- **"取消"**。

### 推送到工具对话框

- 说明：将 Skill 直接复制到工具目录，无需先导入到"我的skills"。
- 以复选框列表展示所有已启用工具，右侧显示推送目录路径。
- 无工具时显示提示信息。
- **"推送到 n 个工具"** 按钮。
- 若存在冲突，后续对话框会列出被跳过的精确 `skill → tool` 配对；**全部覆盖** 只会覆盖这些列出的冲突配对。
- **"取消"**。

### 目录缺失确认

与[推送到工具页](#目录缺失检查)行为相同：在创建缺失推送目录前弹框确认。

### 推送冲突对话框

当 Skill 在目标工具目录中已存在时：

- 列出所有冲突的 Skill 名称。
- **"覆盖全部"**（琥珀色） — `PushStarSkillsToToolsForce()`。
- **"跳过冲突"** — 跳过已存在的，仅推送新的。

## 6. 云端备份

将 Skill 库镜像备份至云端。支持两种后端：**对象存储**（阿里云 OSS、腾讯云 COS、华为云 OBS）和 **Git 仓库**。

### 状态提示

- **云备份未启用提示**（黄色横幅） — 未配置时显示，引导前往设置。

### 操作

| 按钮 | 对象存储标签 | Git 标签 |
|------|------------|---------|
| **立即备份**（Upload 图标） | 立即备份 | 立即备份 |
| **恢复 / 拉取**（Download 图标） | 从云端恢复 | 拉取远端 |
| **刷新**（RefreshCw） | 重新加载最近一次备份的变更结果 | 同左 |

- 云备份未配置时，两个操作按钮均禁用。
- **"备份完成 / Git 同步完成"**（绿色）/ **"备份/同步失败"**（红色）状态消息。

### 本次备份变更列表

- 每次备份或恢复成功后，页面只显示本次操作涉及的文件，不再展示远端全量文件集合。
- 备份页会显示当前应用会话中最近一次成功备份/恢复的**上次同步完成时间**。
- 对象存储与 Git 共用同一种结果列表，按 `新增`、`修改`、`删除` 三种动作展示。
- 已删除文件显示删除标记，不显示文件大小。
- 点击刷新会重新读取当前应用会话里最近一次备份结果。
- 可滚动容器（最大高度限制）。
- **统一备份范围（所有服务商）** — 备份根目录为应用数据根目录（`skills/`、`meta/`、`prompts/`、`config.json` 等）；`cache/` 与 `.git/` 会被排除。
- **对象存储自定义前缀** — 对象存储服务商允许用户填写父级 `remotePath`；SkillFlow 最终总是写入 `<存储桶>/<remotePath>/skillflow/`（若父级路径为空，则为 `<存储桶>/skillflow/`）。
- **云服务商独立配置档案** — 每个云服务商都会保留自己独立的存储桶、路径和凭据配置；在设置中切换服务商时，会恢复该服务商上次保存的表单值，而不会覆盖其他服务商的配置。
- **同步路径可移植** — `meta/*.json`、`star_repos.json` 等同步元数据中的本地路径会以同步根目录下的正斜杠相对路径保存，确保 macOS 与 Windows 间恢复后仍能正确定位。
- **本地路径配置隔离** — `config_local.json` 保存机器相关的文件系统路径与代理设置，例如应用目录外的 `SkillsStorageDir`、工具 `ScanDirs` / `PushDir` 以及代理配置；该文件不参与备份和 git 同步。
- **云端敏感凭据仅本地保存** — Access Key ID、Secret Key、访问令牌等敏感云凭据只会写入 `config_local.json` 中按服务商分组的本地配置；可同步的 `config.json` 仅保留云服务商、存储桶、远端路径、Endpoint、仓库地址、分支等非敏感配置。
- **Git 备份兼容处理** — 当 Git 备份以父目录作为工作树时，SkillFlow 会自动迁移旧的 `skills/.git` 嵌套元数据，避免真实 Skill 文件被当作嵌套仓库而无法跟踪。

### 自动备份

以下操作完成后（在云备份已启用时）自动触发：

- 删除 Skill / 批量删除
- 新建 / 编辑 / 删除提示词
- 手动导入
- 从 GitHub 安装
- 从工具拉取
- 更新 Skill
- 从收藏仓库导入

通过 Wails 事件系统向前端推送进度事件：`backup.started`、`backup.progress`、`backup.completed`、`backup.failed`。

### Git 同步（仅 git 服务商）

选择 **git** 服务商后：

- **仓库自举初始化** — 若 Skills 目录还不是 Git 仓库，SkillFlow 会自动执行初始化，并按配置仓库地址设置 `origin`。
- **远端绑定自动修复** — 若 `origin` 缺失或地址变更，拉取/推送前会自动补齐或更新。
- **启动时自动拉取** — 每次 App 启动时，自动对 Git 备份根目录执行 `git pull`，拉取远端最新内容。
- **远端分支缺失容错** — 首次配置时若远端分支尚未创建，启动拉取会跳过而非直接报错。
- **变更后自动推送** — 与对象存储相同的触发时机；执行 `git add -A && git commit && git push`。
- **定时自动同步** — 由"定时自动同步间隔"设置控制（单位：分钟；0 表示禁用）。后台定时器按配置间隔调用 `autoBackup()`。
- **手动操作冲突检测** — 点击 **立即备份** 与 **拉取远端** 时都会检测 Git 冲突/分叉，若需要用户决策会触发 `git.conflict` 事件。
- **冲突解决对话框** — 若 `git pull` 或 `git push` 检测到冲突或历史分叉，弹出模态框：
  - 对话框会在可用时展示冲突相关文件列表。
  - **"以本地为准"** — 中止合并，将本地状态强制推送到远端。调用 `ResolveGitConflict(true)`。
  - **"以远端为准"** — 中止合并，将本地重置为 `origin/<分支>`。调用 `ResolveGitConflict(false)`。
  - **"手动解决"** — 在系统文件管理器中打开 Git 备份根目录，方便用户直接检查并处理冲突文件。
  - “以本地为准”和“以远端为准”成功后都会从磁盘重载应用状态（skills/meta/config），并发送 `git.sync.completed` 事件。
- **拉取后即时刷新** — 启动自动拉取或手动拉取成功后，会立刻重载磁盘状态，使 `meta/` 与配置变更立即生效。
- 若冲突发生在启动阶段（前端尚未加载），将以待处理标志存储，并在备份页面挂载时通过 `GetGitConflictPending()` 弹出对话框。

## 8. Skill 卡片

在"我的skills"网格和同步页面中使用的可复用卡片组件。

### 两种变体

**主面板卡片**（`SkillCard`）：

| 元素 | 说明 |
|------|------|
| **状态条** | 将来源徽章与其他并存状态（例如可更新 + 已推送工具图标）统一收纳在卡片头部的紧凑区域中；空间足够时优先保持单行，卡片过窄时会自动换行，而不是把状态截断或隐藏 |
| **Skill 名称** | 最多显示两行；保留操作按钮所需间距 |
| **打开目录按钮**（FolderOpenDot，右上角） | `OpenPath(skill.path)` — 在系统文件管理器中打开目录；仅悬停时可见 |
| **选择复选框**（左上角） | 仅多选模式下可见 |
| **悬停操作**（右下角） | 更新（有新版时）· 复制 · 删除 — 悬停前隐藏 |
| **复制按钮** | 读取 `skill.md` 内容，复制到剪贴板，显示"已复制 ✓"持续 2 秒 |
| **拖拽手柄** | 普通模式下可拖拽；拖拽 `skillId` 到侧边栏分类可移动 |
| **右键上下文菜单** | 更新（有新版时）· 移动到 [分类]（每个其他分类一项）· 删除（红色） |

**同步页卡片**（`SyncSkillCard`）：

| 元素 | 说明 |
|------|------|
| **状态条** | 同时显示来源徽章与当前页面允许展示的状态（例如“已导入 / 可更新 / 已推送工具”），并保持在同一块紧凑头部区域内；像“已导入”和“已推送工具”这类并存状态在放得下时会优先同排展示，放不下时再自动换到下一行，确保完整可见 |
| **已推送工具指示器** | 用小尺寸工具品牌图标显示哪些工具的 `PushDir` 已包含该逻辑 skill；不再额外显示箭头前缀，过多时折叠为紧凑计数徽章，并可悬浮查看完整列表 |
| **Skill 名称** | 最多显示两行 |
| **副标题** | 分类名或仓库名 |
| **复制按钮**（悬停） | 剪贴板操作同主面板卡片 |
| **打开目录按钮**（悬停） | 同主面板卡片 |
| **选择复选框**（右下角） | `showSelection = true` 时显示 |

### 统一状态语义

| 状态 | 含义 |
|------|------|
| **installed** | 该逻辑 skill 已经在 **我的skills** 中存在至少一个已安装实例 |
| **imported** | 外部来源页面对 **installed** 的文案别名；在 GitHub / 仓库收藏 / 工具页面中表示“已经在我的skills里” |
| **pushed** | 该逻辑 skill 已经存在于某个工具配置的 **PushDir** 中 |
| **pushedTools** | 哪些工具的 **PushDir** 当前包含该逻辑 skill；卡片用它来渲染工具图标列表 |
| **seenInToolScan** | 该逻辑 skill 已在某个工具配置的 **ScanDirs** 中被扫描到；表示工具里某处已经有它，但不一定是 SkillFlow 推过去的。当前它主要用于分组和关联，不再作为卡片 badge 单独展示。 |
| **updatable** | 某个已安装的 Git Skill，其远端最新 SHA 比本地 `SourceSHA` 更新 |

这些状态由后端基于统一逻辑主键模型计算；前端页面不再独立根据 `Name` 或 `Path` 推断。

```text
                 统一状态语义示意图

[GitHub 候选项] [仓库收藏 Skill] [工具扫描候选项]
        \              |                /
         \             |               /
          +------ 逻辑上是同一个 skill ------+
                         |
                         v
                  [我的skills 实例]
                   installed = true
             外部页面显示 imported = true
                         |
                         +---- 复制到工具 PushDir ----> pushed = true
                         |
                         +---- 远端有更高 SHA -------> updatable = true

[工具 ScanDirs] ---- 扫描到同一逻辑 skill ----> seenInToolScan = true
```

## 9. Skill 悬浮提示

鼠标悬停任意主面板 Skill 卡片 300 毫秒后出现的浮动信息面板。

### 定位规则

- 固定定位，宽 300 px，最大高 400 px。
- 优先显示在卡片右侧；靠近右边界时切换到左侧。
- 靠近底部时向上偏移。

### 内容

| 区域 | 显示字段 |
|------|---------|
| **标题区** | 图标（GitHub / 文件夹）· Skill 名称 · 来源徽章 · 分类 |
| **描述** | 从 `skill.md` YAML 前置元数据解析；无描述时显示"暂无描述"；加载中显示"加载中…" |
| **前置元数据字段** | `argument_hint`（Tag 图标）· `allowed_tools`（Wrench 图标）· `context`（GitBranch 图标） |
| **元数据区** | 仓库 URL（截断，可点击）· 当前 SHA · 可更新 SHA（琥珀色）· 安装日期 · 更新日期 |

## 10. 公共对话框

### 10.1 冲突对话框

推送或拉取时，目标位置已存在同名 Skill，逐条弹出。

- 显示："\[Skill 名称\] 已存在，如何处理？"
- **"跳过"** — 保留现有文件，处理下一个冲突。
- **"覆盖"** — 调用 `*Force` 变体，但只覆盖当前这一个冲突的 skill-target 配对。
- 冲突队列清空后自动关闭。

### 10.2 GitHub 安装对话框

从主面板工具栏打开。

| 控件 | 操作 |
|------|------|
| **URL 输入框** | Git 仓库地址（HTTPS 或 SSH）；回车触发扫描 |
| **"扫描"** 按钮 | `ScanGitHub(url)` — 克隆或拉取仓库，列出 Skill 候选项 |
| **候选项复选框** | 选择要安装的 Skills；已安装的显示徽章 |
| **分类下拉菜单** | 选择目标分类 |
| **"安装 n 个 Skill"** 按钮 | `InstallFromGitHub(url, selected, category)` |

- 提示文字：首次扫描会克隆仓库，后续自动拉取。
- 候选 Skill 会递归扫描整个克隆目录，因此 `plugins/<plugin>/skills/<name>` 这类嵌套布局也会被列出；`skill.md` 文件名按大小写不敏感匹配。
- 递归候选扫描会复用 **设置 → 通用** 中的深度限制（默认 `5`，保存范围 `1-20`）。
- “已安装”徽章按规范化仓库来源 + 子路径解析，而复选框选择状态按候选项路径跟踪，因此同名候选项也能分别处理。
- 候选项行会按照 **设置 → 通用 → 卡片状态显示** 中 GitHub 安装页的策略显示“已导入 / 可更新 / 已推送工具”等状态，并展示候选子路径，避免同名项在对话框中难以区分。
- 分别显示扫描错误和安装错误提示。

### 10.3 目录缺失对话框

推送前，目标目录不存在时弹出。

- 列出每个受影响的工具名称及完整目录路径。
- **"创建并推送"** — 自动创建目录后继续。
- **"取消"** — 中止。

## 11. 后端事件

Go 后端通过 Wails runtime 向前端推送的事件：

| 事件 | 触发时机 | 载荷 |
|------|---------|------|
| `backup.started` | 自动备份开始 | — |
| `backup.progress` | 每上传一个文件 | `{ currentFile: string }` |
| `backup.completed` | 备份完成 | `{ files: Array<{ path, size, action }> }` |
| `backup.failed` | 备份出错 | — |
| `update.available` | 发现 Skill 有新提交 | `{ skillID, skillName, currentSHA, latestSHA }` |
| `star.sync.progress` | 单个仓库同步完成 | `{ repoURL, repoName, syncError }` |
| `star.sync.done` | 所有仓库同步完成 | — |
| `git.sync.started` | Git 拉取/推送开始 | — |
| `git.sync.completed` | Git 同步成功 | 由备份 / 恢复 / 冲突解决触发时为 `{ files: Array<{ path, size, action }> }` |
| `git.sync.failed` | Git 同步出错 | — |
| `git.conflict` | 检测到 Git 合并冲突 | `{ message: string, files?: string[] }` |

主面板监听 `update.available` 事件，实时在对应 Skill 卡片上标记红色更新红点。
备份页监听所有 `git.*` 事件，并在收到 `git.conflict` 时弹出冲突解决对话框。
`App.tsx` 监听全部三个 `app.update.*` 事件，解析发出的 `AppUpdateInfo` payload，并驱动更新对话框的状态机。

## 12. 应用更新对话框

检测到新版本时弹出的居中模态对话框，同时适用于启动时自动检测和设置页手动检测两个入口。对话框由四态状态机驱动：

| 状态 | 触发条件 | 对话框内容 |
|------|---------|----------|
| `available` | 收到 `app.update.available` 事件 | 版本号 + 更新说明 + 平台对应操作按钮 |
| `downloading` | 用户点击"下载并自动重启更新"（仅 Windows） | 旋转图标 + 下载进度提示 |
| `ready_to_restart` | 收到 `app.update.download.done` 事件 | 完成提示 + "立即重启" / "稍后重启"按钮 |
| `download_failed` | 收到 `app.update.download.fail` 事件 | 错误提示 + "前往下载页"按钮 |

### 平台行为

启动检测与手动检测均使用同一模态对话框。

- **Windows** — `available` 状态提供三个选项：
  1. **下载并自动重启更新** — 后台下载新 exe，完成后提示重启。
  2. **前往 Release 页面手动下载** — 在系统浏览器打开 GitHub Releases 页面。
  3. **跳过此版本（下次启动不再提示）** — 记住跳过的版本；启动检测不再对该版本弹窗，手动检测不受影响，仍正常弹出。
- **macOS** — `available` 状态提供两个选项（暂不支持自动下载）：
  1. **前往 Release 页面手动下载** — 在系统浏览器打开 GitHub Releases 页面。
  2. **跳过此版本（下次启动不再提示）** — 与 Windows 行为一致。
- 两个平台都使用同一份 `AppUpdateInfo` payload 渲染当前版本、最新版本和 Release 页面入口。

### 跳过版本机制

- 跳过的版本 tag 存储在 `AppConfig.SkippedUpdateVersion`，并写入共享的 `config.json`，因此重启应用后仍会生效。
- 启动检测时，若 `latestVersion == skippedUpdateVersion`，则**不**发布 `app.update.available` 事件，不弹出对话框。
- 设置页手动点击"检测更新"时，`CheckAppUpdateAndNotify` 始终发布事件，不受跳过版本影响 — 手动检测总是正常弹出对话框。
- 点击"跳过此版本"时调用 `SetSkippedUpdateVersion(latestVersion)`。

### 手动检测按钮（设置页）

设置页标题栏右上角提供 **"检测更新"** 按钮：

- 按钮旁显示当前应用版本号（`vX.Y.Z`）。
- 点击 → 调用 `CheckAppUpdateAndNotify()`；检测中按钮显示旋转图标。
- 若发现新版本，更新对话框通过 `app.update.available` 事件自动弹出。
- 若已是最新版本，内联显示"已是最新版本 (vX.Y.Z)"。
- 出错时内联显示"检测失败: …"。

### 操作说明

| 控件 | 操作 |
|------|------|
| **下载并自动重启更新**（Windows，`available` 状态） | `DownloadAppUpdate(downloadUrl)` — 启动后台异步下载 |
| **前往 Release 页面手动下载**（`available` 状态） | `OpenURL(releaseUrl)` — 在浏览器打开发布页；关闭对话框 |
| **跳过此版本**（`available` 状态） | `SetSkippedUpdateVersion(latestVersion)` — 持久化跳过版本；关闭对话框 |
| **立即重启**（`ready_to_restart` 状态） | `ApplyAppUpdate()` — 写入批处理脚本并退出；脚本完成 exe 替换后重启 |
| **稍后重启**（`ready_to_restart` 状态） | 关闭对话框，不立即重启 |
| **前往下载页**（`download_failed` 状态） | `OpenURL(releaseUrl)` — 在浏览器打开发布页；关闭对话框 |
| **×**（除 `downloading` 外所有状态） | 关闭对话框（本次会话内不再提示） |

### 后端方法

| 方法 | 说明 |
|------|------|
| `GetAppVersion()` | 返回当前版本字符串（CI 通过 `-ldflags` 注入；本地开发为 `"dev"`） |
| `CheckAppUpdate()` | 查询 GitHub Releases API；返回含平台匹配下载链接的 `AppUpdateInfo` |
| `CheckAppUpdateAndNotify()` | 调用 `CheckAppUpdate()` 并在有更新时发布 `EventAppUpdateAvailable`；始终通知，不受跳过版本影响 |
| `GetSkippedUpdateVersion()` | 返回 config 中存储的用户跳过版本 tag |
| `SetSkippedUpdateVersion(version)` | 将跳过版本 tag 持久化到 config；传空字符串可清除 |
| `DownloadAppUpdate(url)` | 异步下载新 exe 到临时目录；完成后发布 `app.update.download.done` 或 `app.update.download.fail` 事件 |
| `ApplyAppUpdate()` | 仅 Windows — 写入批处理脚本实现退出后 exe 替换，然后调用 `os.Exit(0)` |

### 版本注入（CI）

GitHub Actions 在编译时通过 ldflags 注入 tag 名：
```
wails build -ldflags "-X main.Version=${{ github.ref_name }}"
```
当 `Version == "dev"` 时（本地开发），启动时的更新检测会跳过。

## 13. 我的工具

浏览每个已启用工具当前目录中的 Skills。

### 布局

- 左侧边栏列出所有已启用工具。
- 右侧主区域包含一个工具栏，以及 **推送路径** 与 **扫描路径** 两个 Skill 卡片分区。

### 工具栏

| 控件 | 说明 |
|------|------|
| **搜索框** | 实时按名称过滤推送路径与扫描路径中的 Skill 卡片 |
| **排序切换** | 在 **A-Z** 与 **Z-A** 间切换，统一作用于两个分区 |
| **批量删除** | 当当前可见的推送路径列表非空时可进入多选模式 |

### 推送路径分区

- 显示当前工具推送目录中的可删除 Skill。
- 推送路径扫描会复用 **设置 → 通用** 中相同的深度限制（默认 `5`，保存范围 `1-20`）。
- 多选模式下，**全选 / 取消全选** 仅作用于当前可见、已过滤后的推送路径卡片。
- 卡片只显示“已导入 / 可更新 / 已推送到其他工具”等真正有决策价值的状态；其中 `pushed` 使用紧凑的工具图标列表，并会自动排除当前正在查看的工具。

### 扫描路径分区

- 显示仅通过扫描目录发现的只读 Skill。
- 扫描路径发现会复用 **设置 → 通用** 中相同的深度限制（默认 `5`，保存范围 `1-20`）。
- 与推送路径共享同一组搜索和排序控件。
- 扫描路径卡片也使用同一套紧凑状态条，但“被扫描到”这件事由分区本身表达，不再在每张卡片上重复显示 `detected` 徽章。

## 14. 我的提示词

用于在同步的 `prompts/` 目录中保存可复用的 system prompt。

### 导航与存储

- 左侧边栏在 **我的工具** 下方新增 **我的提示词** 入口。
- 每条提示词都保存为备份根目录下的 `prompts/<category>/<name>/system.md`，因此对象存储服务商与 Git 服务商都会自动同步同一批提示词文件。
- 提示词名称为必填项，在整个提示词库内全局唯一，并作为文件夹键名使用。

### 页面布局

- 页面改为和 **我的skills** 一致的左侧分类栏 + 右侧内容区结构。
- 内置 **Default** 作为默认分类。
- 提示词卡片会同时受到分类筛选、关键字搜索和 A-Z / Z-A 排序控制。
- 搜索支持逻辑 `and` / `or` 语法，例如 `review and golang` 或 `summary or changelog`。

### 分类管理

- 可在左侧边栏新建分类。
- 非默认分类支持右键重命名与删除。
- 提示词卡片支持拖拽到分类上移动，交互与 **我的skills** 的分类拖拽一致。

### 提示词卡片

- 卡片复用 Skill 卡片的视觉语言（`card-base`、悬停发光、紧凑操作区）。
- 卡片默认展示 **名称**、可选 **描述** 和内容开头摘要。
- 当内容超出预览范围时，会显示 **点击查看更多** 提示。
- 右上角按钮支持一键复制完整提示词内容到桌面剪贴板，在 Windows 上也会保留多行内容。
- 右下角 **删除** 操作会先弹出确认对话框，再删除卡片和对应的提示词文件夹。

### 提示词编辑器

- 点击 **添加提示词** 会打开内置编辑器，包含 **名称**（必填）、**描述**（可选）、**分类** 和完整 `system.md` 内容。
- 点击已有提示词卡片会打开同一编辑器，并载入现有内容以便编辑和重命名。
- 保存后会写回 `prompts/<category>/<name>/system.md`。

### 导入 / 导出

- 工具栏 **导入** 支持读取 JSON 提示词库文件，并按名称创建或覆盖提示词。
- 工具栏 **导出** 会把完整提示词库写入用户选择的 JSON 文件。
