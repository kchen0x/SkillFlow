# i18n Design ‚Äî Chinese / English Language Support

**Date:** 2026-03-07
**Status:** Approved

## Goal

Add bilingual (zh / en) support to the SkillFlow frontend. All hardcoded Chinese strings are replaced with `t(key)` calls. Users can switch languages via a sidebar button and a Settings > General toggle.

## Architecture

### New files

| File | Purpose |
|------|---------|
| `src/i18n/zh.ts` | All Chinese strings as flat key-value map |
| `src/i18n/en.ts` | All English strings with identical keys |
| `src/i18n/index.ts` | Exports `Lang` union type, `locales` map, `Translations` type |
| `src/contexts/LanguageContext.tsx` | `LanguageProvider` + `useLanguage()` hook |

### LanguageContext API

```ts
type Lang = 'zh' | 'en'
interface LanguageContextValue {
  lang: Lang
  setLang: (lang: Lang) => void
  t: (key: keyof Translations) => string
}
```

- Persisted to `localStorage['skillflow-lang']`, defaults to `'zh'`
- `t(key)` looks up `locales[lang][key]`, falls back to `locales['zh'][key]`

### Translation key structure

Flat dot-notation keys grouped by domain:

- `nav.*` ‚Äî sidebar navigation labels
- `sidebar.*` ‚Äî sidebar non-nav elements (theme button tooltip, feedback)
- `dashboard.*` ‚Äî Dashboard page
- `syncPush.*` ‚Äî SyncPush page
- `syncPull.*` ‚Äî SyncPull page
- `starred.*` ‚Äî StarredRepos page
- `toolSkills.*` ‚Äî ToolSkills page
- `backup.*` ‚Äî Backup page
- `settings.*` ‚Äî Settings page (all tabs)
- `conflict.*` ‚Äî ConflictDialog
- `update.*` ‚Äî Update dialog (in App.tsx)
- `github.*` ‚Äî GitHubInstallDialog
- `category.*` ‚Äî CategoryPanel
- `skillCard.*` ‚Äî SkillCard context menu
- `common.*` ‚Äî shared: save, cancel, delete, loading, confirm, etc.
