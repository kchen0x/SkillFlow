## 完善设计文档与实现计划，新增分类管理和更新检测方法
date: 2026-03-04

### 设计文档更新（skillflow-design.md）
- Go 后端版本明确为 1.26
- `Skill` 结构体新增 `LatestSHA` 字段，用于检测到更新后记录最新 SHA，支持 UI 显示红色角标
- `CloudProvider.Sync` 接口新增 `onProgress` 回调参数，用于向前端报告当前上传文件
- 分类管理行为细化：重命名需同步更新所有 skill 元数据中的 Category 字段；删除时若分类下有 skill，需提示确认并将 skill 移至"未分类"
- Dashboard 左侧分类面板拖拽排序支持持久化到配置

### 实现计划更新（skillflow-implementation.md）
- 新增 `Storage.RenameCategory()`：重命名分类目录并批量更新该分类下所有 skill 的元数据
- 新增 `Storage.DeleteCategory()`：删除前自动将分类下所有 skill 移至未分类
- 新增 `Storage.OverwriteFromDir()`：覆盖安装已有 skill 目录，用于 skill 更新流程
- 新增 `GitHubInstaller.DownloadTo()`：扫描后将 skill 候选项下载到指定目录（导入前的中间步骤）
- 新增 `GitHubInstaller.GetLatestSHA()`：获取 skill 子目录最新 commit SHA，用于更新检测

### 其他
- 新增 `chats/` 目录，用于记录对话历史
