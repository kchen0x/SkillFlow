Project: SkillFlow

Conversation summary
- User reported that on macOS, closing the app did not leave a visible top menu-bar icon and the app could not be restored or fully exited from the menu bar.
- Investigated the existing Wails `HideWindowOnClose` behavior, macOS tray/native status-item integration, and window lifecycle handling.
- Identified that Dock hiding and menu-bar presence were controlled by different layers, and that the original tray initialization/show flow was not robust enough on macOS.
- Iterated on the implementation to make the status item visible, keep the app alive after closing the main window, restore the window safely from the menu bar, and reduce startup stutter.
- Standardized the final macOS approach so Cocoa only manages the menu-bar icon and activation policy while window show/hide goes through the Wails runtime path.
- Reworked the menu-bar icon from plain text to a monochrome brand-like glyph and then tuned it again for a more standard square menu-bar footprint.
- Reviewed the extra debugging logs added during troubleshooting and kept only the lifecycle logs that are useful in production at `info`, while moving detailed diagnostics to `debug`.

Summary of all current uncommitted changes

  - Added native macOS menu-bar status item support with a monochrome template icon, Dock hide-on-close behavior, and menu actions for Show / Hide / Quit.
